## Архитектурное решение

Реализована трёхсервисная архитектура на базе **ASP.NET Core**:

1. **API Gateway** — центральный маршрутизатор, принимающий все клиентские запросы и распределяющий их между микросервисами
2. **File Storing Service** — отвечает за хранение загруженных файлов
3. **File Analysis Service** — выполняет анализ на плагиат и формирует отчеты

## Ключевые элементы реализации

### Коммуникация между сервисами

Используется HTTP-клиент для синхронного взаимодействия. API Gateway регистрирует HttpClient для обоих сервисов:

- FirstService (File Storing) на `http://filestoringservice:8080/`
- SecondService (File Analysis) на `http://fileanalysisservice:8080/`

### Контейнеризация

Все сервисы упакованы в Docker-контейнеры и развертываются через единый `docker-compose.yml`, обеспечивая изолированное окружение и простоту запуска.

### API и документация

Реализована поддержка Swagger для документирования и тестирования API эндпоинтов всех сервисов. Конфигурация включает:

- Swagger документация в режиме разработки
- HTTPS редирект для production
- HSTS политика безопасности

## Антиплагиат
Антиплагиат проверяет, есть ли у нас файл, с абсолютно идентичной последовательностью байт. Отчет выглядит как процент оригинальности файла. 0 - точная копия одного из файлов.
